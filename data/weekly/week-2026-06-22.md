    *   **分析:** 这是一项重要的安全和运维改进。通过支持 OIDC 令牌进行认证，开发者可以利用现有的身份提供商（如 AWS IAM, Google Cloud, GitHub Actions 等）来管理对 Claude API 的访问，从而提高安全性，减少管理长效 API 密钥的风险。
    *   **开发者建议:**

        *   熟悉 OIDC 认证流程，并根据您使用的身份提供商进行配置。
3.  **Claude API Skill 发布:**
    *   **分析:** 这是一个开源的 Agent Skill，为开发者提供了构建基于 Claude 的 Agents 的参考和工具。它包含了 Messages API 和 Claude Managed Agents 的使用指南，并支持多种语言。这降低了开发者利用 Claude 构建复杂 Agent 应用的门槛。
    *   **开发者建议:**
        *   如果您正在开发或计划开发基于 Claude 的自动化 Agent 应用，务必研究这个开源 Skill。
